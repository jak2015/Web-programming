My To-Do List

GET /api/todo - This route will respond with an array listing every ToDoEntry currently stored in memory that has a status of open.
GET /api/todo/completed - This route will respond with an array listing every ToDoEntry currently stored in memory that has a status of completed.
GET /api/todo/:id - This route will respond with a single ToDoEntry that is currently stored in memory.
If the entry is not found, return the following object with a status code of 404, replacing {ID} with the ID you requested: {"error": "An entry with the ID of {ID} could not be found"}
POST /api/todo - This route will create and store a new ToDoEntry; if successful, it will respond with the new ToDoEntry in the response.
PUT /api/todo/:id - This route will update the ToDoEntry with the id of :id and respond with the updated entry.
POST /api/todo/:id/notes - This route will add a note to the ToDoEntry with an id of :id and respond with the updated entry.
DELETE /api/todo/:id - This route will remove the ToDoEntry with the id of :id and respond with the following response: {success: true} If the entry is not found, respond with the following object with a status code of 404, replacing {ID} with the ID you requested: {"error": "An entry with the ID of {ID} could not be found"}
